# Flat Minimal Style

This skill is a preference overlay. Use it with broader design skills such as `design-taste-frontend`, `impeccable`, or project brand guidelines. It defines what "good" should feel like: quiet, exact, structured, self-contained, and informative without noise.

## Design Read

Aim for interfaces that feel:

- calm, not empty
- structured, not boxed in
- minimal, not generic
- premium, not ornamental
- useful, not theatrical
- factual, not impressionistic

The page should feel pulled back. Information should be easy to scan, but the interface should not shout about its structure.

## Core Principles

1. Prefer quiet structure over visible structure.
2. Use spacing, alignment, pale surfaces, and typography before borders, dividers, shadows, or accent color.
3. Create outline-less enclaves when grouped content needs containment.
4. Keep explanatory copy narrow for readability.
5. Let key-point regions, examples, asset grids, and grouped rules break wider than the reading column.
6. Balance breathing room with information density. Avoid both cramped UI and empty gallery spacing.
7. Preserve factual visual fidelity: logo geometry, optical alignment, font, downloadable assets, motion, and rendered examples must match.
8. For external-facing surfaces, remove internal product concepts unless they help outsiders use the brand or product.

## Outline-Less Enclaves

An enclave is a soft, self-contained region without a visible outline. Use it when content should feel grouped, calm, and protected.

Use:

- pale warm surface slightly distinct from the page
- no border
- no heavy shadow
- generous internal padding
- enough outer whitespace for the region to read as intentional
- compact heading and body hierarchy

Avoid:

- cards so faint they disappear
- borders added just to prove containment
- shadow used as decoration
- nesting cards inside cards
- equal spacing everywhere

Good proportions:

- two-column naming or rule cards can be wide and calm
- three-column key-point cards may break out wider than the main text column
- card padding should feel generous, but not like empty gallery space
- body copy should remain readable and direct

## Width Strategy

Use two widths:

- Reading column: section headers, intros, explanatory copy.
- Breakout width: key points, naming rules, logo assets, downloads, usage examples, comparisons, and reusable rules.

Do not keep everything in one narrow column if it makes important groups feel squeezed. Breaking width is appropriate when it creates a calmer, more self-contained field.

## Spacing Rhythm

Related items should be close. Distinct groups should have visible air between them.

Default rhythm:

- tight within heading groups
- medium gap between header and content
- generous gap between sections
- wider gaps between cards when cards have substantial copy
- slightly smaller gaps when cards are mostly visual assets

Do not solve tightness by adding space everywhere. Tune the region that feels cramped.

## Typography

Use type as the primary hierarchy tool.

- Prefer larger, readable body copy in key cards.
- Use weight and spacing before decorative color.
- Keep labels compact and direct.
- Avoid inflated AI or SaaS language.
- Use plain product and brand wording.

## Brand Fidelity

Small visual inaccuracies matter. Always inspect:

- logo and wordmark optical alignment
- mark size and baseline relationship
- actual brand font or nearest factual substitute
- displayed asset versus downloadable asset
- mobile rendering
- nav and footer consistency across pages
- hover and motion behavior

If the page displays a brand asset and also offers it for download, they must match.

## External-Facing Filter

For public brand kits, docs pages, and marketing surfaces:

- keep only sections an outsider can use
- remove internal examples that require product context
- prefer "how to use the brand" over "how the product works"
- make downloads and naming rules obvious
- make misuse examples clean and immediately understandable

## Review Checklist

Before finishing, answer:

- Does each grouped region feel self-contained without needing a border?
- Is there enough air around enclaves for them to feel intentional?
- Did key regions break width where helpful?
- Is information still easy to scan?
- Is the page calm without becoming empty?
- Are assets, fonts, and downloadable files factually aligned?
- Does mobile feel composed rather than merely collapsed?
